# RP2350 無人機活塞引擎 EFI 韌體開發技術手冊

## 引擎電控單元（ECU）與電子燃油噴射（EFI）─ 四衝程 / 二衝程 UAV 雙版本

> 文件版本：2026 年 4 月　｜　硬體平台：Raspberry Pi RP2350（Curio RB-RP2354A）  
> 技術範疇：嵌入式韌體 · 感測器融合 · 卡爾曼濾波 · 無人機活塞引擎控制

---

## 1．專案背景與設計目標

本專案以 **Raspberry Pi RP2350**（雙核 Cortex-M33，150 MHz）為核心，實作一套完整的開源 EFI（Electronic Fuel Injection）韌體，專為無人機（UAV）活塞引擎設計。專案提供**四衝程**與**二衝程**兩個獨立韌體版本，均具備以下設計目標：

- **即時確定性**：核心控制迴路（Core 0）以 2 kHz 固定頻率運行，噴油與點火排程精度達微秒級
- **高空適應性**：透過氣壓感測器（MS5611 BAP）動態修正空氣密度，支援至海拔 5,000 m 飛行
- **失效維持（Fail-Functional）**：單一感測器故障時以預設安全值繼續供油，不停機
- **開源可擴充**：所有查找表（VE、AFR、Alpha-N、點火提前角）均可透過 UART 串口即時更新

## 2．系統架構概覽

### 2.1 雙核心工作分配

RP2350 的兩個 Cortex-M33 核心按確定性原則嚴格分工：

| 核心 | 職責 | 執行頻率 |
|------|------|----------|
| **Core 0** | ADC 讀取 → 卡爾曼濾波 → 噴油計算 → 噴油嘴排程 → 點火排程 → IAC PID | **2,000 Hz（每 500 µs）** |
| **Core 1** | UART CSV 輸出 → DroneCAN 遙測 → 串口調校指令接收 | 100 Hz（每 10 ms） |

雙核心共享資料以 **Seqlock（序列鎖）** 保護，搭配 ARM DMB 記憶體屏障防止資料撕裂：

```
Core 0 寫入：seq++（奇數）→ 寫資料 → DMB → seq++（偶數）
Core 1 讀取：s = seq → 讀資料 → DMB → 驗證 seq 未變且為偶數，否則重試
```

### 2.2 專案檔案結構


## 3．感測器配置與接線

### 3.1 接腳對照表

| 接腳 | GPIO | 功能 | 備註 |
|------|------|------|------|
| CKP | GP2 | 曲軸位置（PIO SM0） | 60-2 齒輪（四衝程）/ 12-1（二衝程） |
| CMP | GP3 | 凸輪位置（PIO SM1） | **僅四衝程使用** |
| INJ1 / INJ2 | GP6 / GP7 | 噴油嘴 MOSFET | 開集極驅動，主動低 |
| IGN1 / IGN2 | GP10 / GP11 | 點火線圈 IGBT | 高電位充電，下降沿觸發 |
| IAC | GP14 | 怠速空氣閥 PWM | 50 Hz，16-bit 解析度 |
| FUEL_PUMP | GP15 | 燃油泵繼電器 | 12 V 線圈 |
| CHOKE | GP16 | 阻風門電磁閥 | EGT 緊急補濃 |
| ENGINE_KILL | GP17 | 熄火繼電器 | 最後手段 |
| **OIL_INJ** | **GP13** | **機油噴射泵 PWM** | **僅二衝程使用** |
| MAP | GP26 (ADC0) | 進氣岐管壓力 | MPX4250A，0-250 kPa |
| TPS | GP27 (ADC1) | 節氣門位置 | 分壓至 3.3 V |
| IAT | GP28 (ADC2) | 進氣溫度 NTC | 10 kΩ 熱敏電阻 |
| CHT | GP29 (ADC3) | 汽缸頭溫度 NTC | 取代車規冷卻水溫 |
| BAP (CS) | GP21 / SPI0 | 大氣壓力 MS5611 | 精度 ±1.5 Pa |
| EGT (CS) | GP22 / SPI0 | 排氣溫度 MAX31855 | K 型熱電偶，±2°C |
| CAN TX/RX | GP23 / GP24 | DroneCAN 遙測 | UAVCAN v1 |
| UART TX/RX | GP4 / GP5 | 串口調校 | 115,200 baud |

### 3.2 NTC 熱敏電阻轉換（Steinhart-Hart β 模型）

```
R_ntc = R_pullup × V_adc / (V_ref - V_adc)
1/T   = 1/T_ref + (1/β) × ln(R_ntc / R_ref)
```

- 參數：R_ref = 10 kΩ @ 25°C，β = 3977，上拉電阻 R_pullup = 2.49 kΩ

## 4．感測器融合：卡爾曼濾波器

### 4.1 設計動機

原始 ADC 讀值存在以下問題，需要濾波：

| 感測器 | 雜訊來源 | 原始 RMS 雜訊 | KF 後 RMS 雜訊 |
|--------|----------|--------------|----------------|
| MAP（ADC） | 電氣干擾、引擎振動 | ~2 kPa | < 0.5 kPa |
| IAT（NTC） | 熱敏電阻量化雜訊 | ~2°C | < 0.3°C |
| EGT（MAX31855） | 放大器熱雜訊 | ~3°C | < 0.8°C |
| BAP（MS5611） | 感測器固有雜訊 | ~0.3 kPa | < 0.1 kPa |

### 4.2 MAP + BAP 融合卡爾曼濾波器（3 狀態）

這是本系統最關鍵的創新：將進氣岐管壓力與大氣壓力同時納入同一個 KF，自動維護高度基準：

```
狀態向量：x = [ P_manifold,  dP/dt,  P_baro ]ᵀ

狀態轉移矩陣 F（dt = 0.5 ms）：
    ┌ 1  dt  0 ┐
F = │ 0   1  0 │
    └ 0   0  1 ┘

量測矩陣 H（MAP ADC + MS5611）：
    ┌ 1  0  0 ┐     z₁ = MAP（ADC 量測）
H = └ 0  0  1 ┘     z₂ = BAP（MS5611 量測）

過程噪聲 Q = diag(0.2, 2.0, 0.001)  [kPa², (kPa/s)², kPa²]
量測噪聲 R = diag(4.0, 0.09)        [kPa², kPa²]
```

**關鍵效益**：
- `dP/dt` 狀態提供加速補油的直接觸發信號，無需額外微分器
- 當 MAP 感測器故障時，BAP 狀態繼續由 MS5611 更新，維持大氣壓力估計
- MAP 相對壓力 = `P_manifold - P_baro` 在任何高度下均能正確反映引擎負載

### 4.3 EGT 卡爾曼濾波器（1 狀態）

```
狀態：x = [T_exhaust]，Q = 100.0，R = 9.0
```

Q 值較大（100）因為 EGT 在急加速時可瞬變 100°C/s，需要快速跟蹤。

## 5．燃油計算邏輯

### 5.1 IPW 計算瀑布

```
m_air  = ρ × V_swept × VE%          理想氣體估算空氣質量
         ρ = P_man[Pa] / (287.05 × T[K])

IPW_SD = (m_air / AFR_target) / inj_flow    Speed-Density 基礎脈寬
IPW_AN = table_lookup(RPM, TPS%)             Alpha-N 備援脈寬
IPW_blend = α×IPW_AN + (1-α)×IPW_SD         動態混合
IPW_wu    = IPW_blend × warm_up(CHT)         冷啟動加濃
IPW_accel = IPW_wu + accel_decay             加速補油
IPW_EGT   = IPW_accel × egt_factor(EGT)     EGT 保護補濃
IPW_final = IPW_EGT × (1+λ_trim) + dead_time 閉迴路修正 + 死區時間
```

### 5.2 Alpha-N / Speed-Density 動態混合

```
混合因子 α = max( f(RPM),  f(|dP/dt|) )

f(RPM)：
  RPM < 2000 → α = 1.0（全 Alpha-N）
  RPM > 3500 → α = 0.0（全 Speed-Density）
  中間：線性插值

f(|dP/dt|)：
  |dP/dt| > 20 kPa/s → α = 1.0
  |dP/dt| < 0        → α = 0.0
```

### 5.3 EGT 三段保護

| 階段 | EGT 範圍 | ECU 動作 |
|------|----------|----------|
| 正常 | < 警告門檻 | 正常閉迴路控制 |
| 警告 | 警告 ~ 緊急 | 每超過 50°C 加濃特定百分比，停止閉迴路 |
| 緊急 | > 緊急門檻 | 最大加濃，啟動阻風門電磁閥，DroneCAN 告警 |

四衝程門檻：警告 750°C / 緊急 850°C，加濃 5%/50°C  
二衝程門檻：警告 680°C / 緊急 780°C，加濃 7%/50°C（更保守）

## 6．點火系統

### 6.1 四衝程點火時序

```
點火週期：720°（兩圈）
Cyl1 TDC：  0°    Cyl2 TDC：360°

線圈充電開始角 = TDC - advance - dwell
線圈充電角度上限：60°（7000 RPM 以內充電時間充裕）
最大提前角：35° BTDC
```

### 6.2 二衝程點火時序

```
點火週期：360°（一圈）
Cyl1 TDC：  0°    Cyl2 TDC：180°（平衡雙缸）

線圈充電角度上限：40°（← 比四衝程更短，充電窗口減半）
最大提前角：28° BTDC（← 更保守，防止震爆與拉缸）
```

### 6.3 點火提前角查找表（二衝程特徵）

二衝程點火提前角圖的特徵與四衝程截然不同──呈現一個以排氣管共振 RPM（約 4,000 RPM）為頂點的「丘形」：

```
提前角
 30°│                    ▲ 4000 RPM（排氣共振峰值）
    │                 ╱     ╲
 20°│              ╱           ╲
    │           ╱                 ╲
 10°│        ╱                       ╲
    │     ╱                             ╲
  0°└────────────────────────────────────── RPM
    800        3000     4000     6000     9000
```

### 6.4 硬體鬧鐘 ISR 排程

RP2350 的 `hardware_alarm`（定時器）以微秒精度觸發線圈充電與觸發回調：

```c
// 計算從當前曲軸角到目標角的時間偏移量
t_offset_us = (delta_angle / 360.0f) * (60.0e6f / RPM);
hardware_alarm_set_target(alarm_id, time_us_64() + t_offset_us);
```

## 7．四衝程與二衝程核心差異對照

| 項目 | 四衝程 | 二衝程 |
|------|--------|--------|
| **動力衝程** | 每 720°（兩圈） | 每 360°（一圈） |
| **凸輪感測器** | 需要（區分壓縮與排氣 TDC） | **不需要** |
| **觸發輪** | 60-2 齒 | 12-1 齒（更簡單） |
| **噴油頻率** | 每 720° 噴一次 | 每 360° 噴一次 |
| **線圈充電上限** | 60°（2.5 ms @ 典型 RPM） | 40°（1.8 ms，充電窗口減半） |
| **最大點火提前角** | 35° BTDC | 28° BTDC（震爆敏感性更高） |
| **VE 曲線形狀** | 平滑丘形（廣義高效區） | 尖峰丘形（排氣共振 ~4000 RPM 處 VE > 100%） |
| **EGT 警告門檻** | 750°C / 850°C | 680°C / 780°C（更低，更敏感） |
| **潤滑方式** | 獨立機油池（不需額外控制） | **機油噴射泵（GP13 PWM）** |
| **目標空燃比** | 14.7:1 巡航 / 12.0 WOT | 13.5:1 巡航 / 11.0 WOT（略偏富油） |

### 7.1 二衝程專用機油噴射控制

二衝程引擎採用「油氣分離」設計，需要獨立的機油噴射泵（GP13，PWM 50 Hz）：

```c
// 機油泵占空比：隨 RPM 增加，輕負載時略多（保護活塞環）
duty_base   = 20% + (RPM / 9000) × 40%     // 20-60%
duty_factor = 1 + (1 - TPS%) × 0.3          // 輕負載補償
duty_final  = clamp(duty_base × factor, 10%, 80%)
```

等效油氣比：怠速 ~33:1（3% 機油），全油門 ~50:1（2% 機油）

## 8．高空補償機制

### 8.1 大氣壓力→海拔（對流層等溫式）

```
H = 44,330 × [1 - (P_bap / 101,325)^0.1903]     [公尺]
```

### 8.2 空氣密度比 σ

```
σ = (P_bap / 101,325) × (288.15 / T_iat[K])

在 3,000 m ISA 標準條件下：σ ≈ 0.74（空氣密度降低 26%）
```

### 8.3 Speed-Density 空氣質量（含海拔效應）

```
P_man 本身已反映高空進氣壓力，以下公式自動包含海拔影響：

ρ = P_manifold[Pa] / (287.05 × T_air[K])
m_air = ρ × V_swept × VE%
```

MAP 感測器失效時改用備援模式：

```
m_air_failsafe = σ × VE% × m_ref_sea_level
```

## 9．失效維持策略（Fail-Functional）

UAV 的核心哲學：**引擎停止 = 墜機，因此任何單一感測器故障均不得停機**。

| 故障類型 | 汽車跛行模式 | UAV 失效維持模式 |
|----------|-------------|----------------|
| MAP 感測器斷線 | 限制轉速 | 以 BAP × 0.75 替代，繼續供油，標記 `FF_MAP_FIXED` |
| IAT 感測器斷線 | 警告燈 | 以 20°C 預設值繼續計算，標記 `FF_IAT_FIXED` |
| EGT 超過警告值 | 可能切斷引擎 | 強制補濃，停止閉迴路修正，DroneCAN 告警 |
| EGT 超過緊急值 | 切斷引擎 | 最大補濃，啟動阻風門電磁閥，**不切斷引擎** |
| Core 0 超時 | 停機 | 設定 `FAULT_CORE_OVERRUN` 旗標，繼續運行 |

故障旗標為 16 位元位元遮罩（`fault_flags`），透過 DroneCAN 實時傳送至地面站。

```bash
cd rp2350_efi_4stroke
```

**四衝程 CSV 欄位**：
```
RPM, MAP_kPa, BAP_kPa, 高度_m, IAT_°C, EGT_°C, Lambda, IPW_µs,
密度比, 點火提前角, λ_trim, 故障旗標
```

**二衝程 CSV 額外欄位**（最後兩欄）：
```
..., λ_trim, 機油泵占空比_%, 故障旗標
```

## 12．進階開發方向

本韌體已實作完整的基礎 EFI 架構，未來可在此基礎上進一步發展：

- **強化學習（RL）自動標定**：以 Python 離線訓練 VE 表修正策略，部署為 C++ 查找表更新
- **自適應 KF 噪聲估計**：在飛行中自動調整 Q/R 矩陣，適應不同海拔與溫度條件
- **雙點火冗餘系統**：仿照航空慣例，增加第二套獨立點火線圈與驅動電路
- **飛行資料記錄（FDR Lite）**：將 ECU 輸出同步寫入 SD 卡，支援事後診斷與 VE 表優化
- **DroneCAN / UAVCAN v1 完整實作**：整合 cyphal/libuavcan 堆疊，支援引擎狀態廣播與遠程調校
